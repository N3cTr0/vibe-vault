# Network (page)

Card order (0.19.0): Network Tools → Ping & Diagnostic Tools → **IP Scanner** → Wi-Fi → Active Connections → per-adapter cards → **the two destructive resets last** (risk increases as you scroll, same convention as Repair).

- **Network Tools** (no input needed; output box appears on first use): Flush DNS, Release/Renew IP (**gated** — drops connectivity, can cut a remote session), Show Public IP (api.ipify.org), Connectivity Test (gateway + 1.1.1.1 + 8.8.8.8 + DNS resolve), Speed Test (25 MB Cloudflare download, single stream, rough Mbps). The output log **clears at the start of each command** (`NetGuarded`) so results don't pile up confusingly (0.17.55). (The separate Ping/Diagnostic Tools box already replaces its text per run.)
- **Wi-Fi:** connected/not-connected only (by design — SSID needs Location consent on Win11, see [[WPF Single-File Quirks]]); saved profiles with **Show password** (confirmed + activity-logged; `netsh wlan … key=clear`); **Scan Nearby** button (moved here 0.17.65 — it never needed a host) lists nearby networks in a "Nearby Networks" section, hidden until scanned.
- **Network Stack Reset** (winsock + int ip reset + flushdns; restart required) and **Full Network Reset** (`netcfg -d`, reinstalls adapters on reboot; clears VPN clients/static IPs/saved Wi-Fi) — both tech-gated + strongly confirmed.
- **Ping & Diagnostic Tools** (redesigned 0.17.65): one aligned input row — Host/IP (stretching) + compact DNS type + Port — above a uniform 4-button row (Ping, Traceroute, DNS Lookup, Port Check); output box appears on use. **Port Check does multiple ports (0.19.4):** `80,443,3389` or a range `1000-1010` (`NetTools.ParsePorts` → `PortCheckManyAsync`), one line per port, and it distinguishes **OPEN / CLOSED (refused) / FILTERED (dropped)** — the old version only checked `client.Connected`, so refused and timed-out looked the same.
- **IP Scanner** (0.19.0) — Advanced IP Scanner-style range sweep, built into the page (not a new tab); engine is `IpScanner.cs`. Input accepts **CIDR** (`192.168.1.0/24`), a **span** (`192.168.1.1-254` or full `a-b`) or a **single IP**, prefilled with the machine's own /24 (`DefaultRange()`). Each address: **ICMP ping** → **ARP** (`SendARP` via iphlpapi — catches ping-blocking hosts on our own segment + returns the MAC) → **reverse DNS**. **Deep scan** additionally TCP-knocks 445/3389/139/80/443/22 (the only way to see a ping-blocking host across a router, since ARP is link-local). 64-wide `SemaphoreSlim`, streams rows in numeric order via `ObservableCollection` + insertion sort, live progress bar, **Stop** (CancellationToken), **Copy** to clipboard. **Deep scan lists every open port per host (0.19.4)** in an **Open ports** column — it knocks all `DeepPorts` concurrently (`ScanPortsAsync`) on every live host, not just the first hit / ping-blockers. Marks "this PC"/"gateway". Read-only; capped at `MaxHosts = 8192`. Addresses are held as host-order `uint` internally (so they sort/increment naturally); `SendARP` needs network byte order (`ToNetwork`). The scan is activity-logged. **Note:** couldn't be exercised headless during dev — the safety classifier blocks network-probing binaries and ASR blocks exe launches from Temp — so it was verified by compile + logic review; real testing happens in the app.
- **Active connections & listening ports** (netstat-style with owning process) and per-adapter cards (IP/mask/gateway/DNS/MAC, DHCP lease).
