# 🧠 Vibe Projects — Home

The vault is organised with the **PARA method**: Projects (active work), Areas (ongoing responsibilities), Resources (reference material), Archive (done/inactive).

## Projects

- [[PartnerTool]] — Progressive Computing's Windows support/diagnostic tool *(the project hub — start here)*
- [[Claude Dev VM]] — dedicated Win11 VM (VMware Workstation Pro) giving Claude Code full autonomous access for development

## How this vault works

- Every project gets a folder under `01 Projects` and a **hub note (MOC)** named after the project — that hub is the reference point; link everything to it.
- Notes carry `project:` frontmatter so cross-project searches stay clean.
- New knowledge from work sessions gets captured here going forward — decisions, gotchas, fixes, and code snapshots — so nothing lives only in a chat history.
